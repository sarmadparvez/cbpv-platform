Welcome to Crowd-Based Prototype Validation Platform

You have Developer and Crowd-Worker roles.

  • You can create Prototypes as a Developer to get feedback on them.
  • You can also give feedback on other Prototypes as a Crowdworker.

  • You can create Prototypes as a Developer to get feedback on them.

You have Crowd-Worker role.

  • You can give feedback on other Prototypes as a Crowdworker.

On this platform, developers can present their application ideas as a prototype and crowd-workers evaluate those prototypes. The evaluation results are presented to developers and crowd-workers receive incentives from them.

As a Developer you start by creating a Project. A Project is created for each application idea for which you want to have feedback on its prototype. A Project contains Tasks and each Task is an iteration for the Prototype. Inside a Task, you can add your prototype for evaluation. To do that, you start new iteration by by providing basic Task details such as Title and Description. This title and description will also be shown to crowd-works when they will evaluate your prototype so you can provide any relevant details for them in the description field. When starting a new iteration you also need to select Prototype Format. Currently, three prototype formats are supported i.e Image, Link to the Prototype, or Textual description of application idea. Then you need to select the Test Type for the evaluation i.e either Basic Test or Comparison Test. In a basic test, a single prototype is evaluated, whereas in a comparison test two prototypes of the same Application are evaluated with a comparison. Then you also need to specify Crowd Matching Criteria which is a filter for crowd-workers and restricts the prototype to be evaluated by only those crowd-workers who meet these criteria. At this point you also specify Incentive and total budget for this Task iteration. With all of this detail you can create a Task and the Task will be created in Draft state by default. A Task in the draft state is not visible to crowd-workers. After that, you need to add your prototype to the Task in the selected format, and add a Questionnaire. In this questionnaire add all the questions about the prototype. This questionnaire along with the task title, description, and prototype will be presented to crowd-workers when they evaluate your prototype. Finally after entering all the data and providing the required information you can Activate the Task. Once activated, the Task can no longer be edited and it will be immediately available for crowd-works to provide feedback. You can see the individual feedback and aggregated feedback once the feedback is provided by crowd-workers. After that, you can close the Task iteration and start a new iteration which will also follow the same flow as explained above.

As a Crowd-Worker you can Evaluate prototypes. The prototypes which you can evaluate are shown as Open Tasks. Only those Tasks are shown to you for which your profile matches the specified criteria by the developer (Task owner). If you cannot see open Tasks then this could be due to two reasons. Either there is no open Task, or otherwise, your profile does not match any Task's evaluation criteria. You can also filter the Tasks in the Task list e.g by skills, country, prototype format, test type, task name, etc. To evaluate a prototype you can open the Task and there you can see all relevant Task details and the prototype. To provide feedback you need to answer questions in the Questionnaire for that Task. There you can also provide additional feedback or comment related to the prototype. Once submitted, the feedback cannot be edited anymore and will be available to developers. You can also see all of your provided feedbacks.
